### 发布者-事件总线-订阅者 示例
下面给出修改后的代码，将原来的观察者模式重构为**发布者-事件总线-订阅者**架构，并使用多文件组织。新的设计包含一个独立的事件总线（`EventBus`），多个发布者（`NewsAgency`、`SocialMedia`）和一个订阅者（`User`），所有通信通过总线进行，实现完全解耦。

#### 1. `News.h` —— 新闻数据定义

```cpp
#pragma once
#include <string>
#include <iostream>
#include <ctime>

struct News {
    std::string title;
    std::string content;
    std::string author;
    std::string timestamp;

    News(const std::string& t, const std::string& c, const std::string& a)
        : title(t), content(c), author(a) {
        time_t now = time(0);
        timestamp = ctime(&now);
    }

    void printNews() const {
        std::cout << "Title: " << title << "\n"
                  << "Author: " << author << "\n"
                  << "Published at: " << timestamp
                  << "Content: " << content << std::endl;
    }
};
```

---

#### 2. `EventBus.h` —— 事件总线声明

```cpp
#pragma once
#include <functional>
#include <vector>
#include "News.h"

class EventBus {
public:
    using NewsHandler = std::function<void(const News&)>;

    // 订阅新闻事件
    void subscribe(NewsHandler handler);

    // 发布新闻事件（通知所有订阅者）
    void publish(const News& news);

private:
    std::vector<NewsHandler> handlers_;  // 所有新闻订阅者
};
```

---

#### 3. `EventBus.cpp` —— 事件总线实现

```cpp
#include "EventBus.h"
#include <iostream>

void EventBus::subscribe(NewsHandler handler) {
    handlers_.push_back(handler);
}

void EventBus::publish(const News& news) {
    std::cout << "[EventBus] Publishing news to all subscribers..." << std::endl;
    for (const auto& handler : handlers_) {
        handler(news);
    }
}
```

#### 4. `Publisher.h` —— 发布者基类（可选）和具体发布者声明

```cpp
#pragma once
#include "EventBus.h"
#include "News.h"

// 新闻社发布者
class NewsAgency {
public:
    explicit NewsAgency(EventBus& bus);
    void publishNews(const std::string& title, const std::string& content, const std::string& author);

private:
    EventBus& bus_;
};

// 社交媒体发布者（第二个发布者示例）
class SocialMedia {
public:
    explicit SocialMedia(EventBus& bus);
    void postNews(const std::string& title, const std::string& content, const std::string& author);

private:
    EventBus& bus_;
};
```

---

#### 5. `Publisher.cpp` —— 发布者实现

```cpp
#include "Publisher.h"

NewsAgency::NewsAgency(EventBus& bus) : bus_(bus) {}

void NewsAgency::publishNews(const std::string& title, const std::string& content, const std::string& author) {
    News news(title, content, author);
    bus_.publish(news);
}

SocialMedia::SocialMedia(EventBus& bus) : bus_(bus) {}

void SocialMedia::postNews(const std::string& title, const std::string& content, const std::string& author) {
    News news(title, content, author);
    bus_.publish(news);
}
```

#### 6. `User.h` —— 订阅者声明

```cpp
#pragma once
#include <string>
#include "News.h"

class User {
public:
    explicit User(const std::string& name);
    void onNewsReceived(const News& news);   // 回调函数

private:
    std::string name_;
};
```

---

#### 7. `User.cpp` —— 订阅者实现

```cpp
#include "User.h"
#include <iostream>

User::User(const std::string& name) : name_(name) {}

void User::onNewsReceived(const News& news) {
    std::cout << name_ << " received the news:\n";
    news.printNews();
}
```

---

#### 8. `main.cpp` —— 主程序

```cpp
#include <functional>
#include "EventBus.h"
#include "Publisher.h"
#include "User.h"

int main() {
    // 创建事件总线（全局通信中枢）
    EventBus bus;

    // 创建发布者（多个）
    NewsAgency agency(bus);
    SocialMedia social(bus);

    // 创建订阅者（用户）
    User alice("Alice");
    User bob("Bob");

    // 用户通过总线订阅新闻事件
    bus.subscribe(std::bind(&User::onNewsReceived, &alice, std::placeholders::_1));
    bus.subscribe(std::bind(&User::onNewsReceived, &bob, std::placeholders::_1));

    // 发布者发布新闻（经过总线转发）
    agency.publishNews("C++20 Released", "C++20 introduces modules, coroutines, and more.", "ISO Committee");
    social.postNews("Weather Update", "Sunny with a chance of rain.", "Weather Channel");

    return 0;
}
```

#### 设计说明

1. **事件总线（EventBus）**：不再属于任何特定模块，它是一个独立的全局组件，负责维护订阅者列表并转发事件。
2. **发布者（NewsAgency、SocialMedia）**：它们只知道事件总线，发布时只需调用 `bus.publish()`，完全不知道有哪些订阅者存在。
3. **订阅者（User）**：通过总线订阅事件，回调函数会在事件发生时被总线调用。
4. **解耦**：发布者和订阅者之间没有直接依赖，新增发布者或订阅者都不需要修改对方代码。

这个架构完美体现了**发布者-事件总线-订阅者**的运行关系，且符合你的引擎框架图中事件总线的定位。
